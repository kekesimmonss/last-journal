CS 230 Project Reflection

The Gaming Room was the client for this project. They wanted to expand their existing Android game, Draw It or Lose It, into a cross-platform application that could be accessed by users on Windows, macOS, Linux, Android, and iOS. Their primary requirement was a web-based distributed application that would support multiple players, teams, and games while providing secure authentication, reliable performance, and the ability to scale as the number of users increased. To meet these requirements, I designed a three-tier architecture with separate presentation, application, and data layers to ensure flexibility, maintainability, and platform independence.

One aspect of this project that I completed particularly well was evaluating different operating platforms and selecting the most appropriate solution based on the client's requirements. I carefully compared Linux, Windows, macOS, and mobile devices by considering factors such as performance, scalability, security, cost, and compatibility. This evaluation helped me justify recommending Linux as the primary server platform because of its stability, open-source nature, and strong support from cloud providers. Working through the design document before writing code was also very helpful because it required me to organize my ideas, identify system requirements, and think about the architecture before implementation. Creating the design first made the development process more organized and reduced the likelihood of major design changes later.

If I could revise one part of my work, I would expand the system architecture section by including detailed diagrams showing the interactions between clients, servers, databases, and cloud services. Visual representations would make the design easier to understand and better communicate how the different components work together. Throughout the project, I interpreted the user's needs by focusing on scalability, security, cross-platform compatibility, and ease of maintenance. Considering user needs is essential because software should solve the client's problem while providing a reliable and intuitive experience for end users. In future projects, I will continue using a structured design approach by gathering requirements first, comparing possible solutions, creating architectural models, and evaluating trade-offs before beginning implementation. This process helps produce software that is organized, maintainable, and aligned with both business and technical goals.
